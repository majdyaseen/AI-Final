Ch.5 Adversarial Search & Games

Game Types

AxisOptions
Deterministic vs StochasticChess (det.) vs. Backgammon (stoch.)
Perfect vs Imperfect InfoChess (perfect) vs. Poker, Kriegspiel (imperfect)
Zero-sumOne player's gain = other's loss
Players2-player, multi-player

Minimax

  • MAX player maximizes utility; MIN player minimizes it.
  • Recursively compute minimax value of each node.
  • Complete & optimal vs. rational opponent.
  • Time: O(b^m), Space: O(bm).
  • Multiplayer: Each node stores a vector of utilities (one per player); each player maximizes their own component.
MINIMAX(state): if TERMINAL(state): return UTILITY(state) if MAX's turn: return max over actions of MINIMAX(RESULT(state,a)) if MIN's turn: return min over actions of MINIMAX(RESULT(state,a))

Alpha-Beta Pruning

  • Prunes branches that cannot affect the final decision.
  • alpha: best value MAX can guarantee so far (starts -inf).
  • beta: best value MIN can guarantee so far (starts +inf).
  • Prune when alpha ≥ beta.
  • Best case (perfect ordering): O(b^(m/2)) — effectively doubles search depth.
  • Average (random ordering): O(b^(3m/4)).
Key insight: A MIN node below a MAX node is pruned if its value falls below alpha. A MAX node below a MIN node is pruned if its value exceeds beta.

Heuristic Minimax (H-MINIMAX)

  • Cuts off search at depth d and applies evaluation function EVAL(s).
  • EVAL must: (1) be fast, (2) correlate with true utility, (3) agree on terminals.
  • Quiescence search: Extend search beyond cutoff when position is "noisy" (captures, checks).
  • Horizon effect: Agent pushes bad moves beyond search depth — use singular extensions to avoid.
  • Forward pruning: Beam search, killer moves, futility pruning — prune unlikely branches early.
  • Lookup tables: Pre-computed endgame values (e.g., all 5-piece chess endgames).

Monte Carlo Tree Search (MCTS)

  • Does not require an evaluation function — uses random playouts (simulations).
  • 4 phases per iteration:
PhaseDescription
1. SelectionFollow UCB1 policy from root to leaf
2. ExpansionAdd one or more child nodes
3. SimulationRandom playout to terminal state
4. BackpropagationUpdate win/visit counts up the tree
  • UCB1 formula: UCB1(n) = U(n)/N(n) + C * sqrt(log N(Parent)/N(n))
  • C controls exploration vs. exploitation tradeoff.
  • Anytime algorithm — can stop at any point and return best move so far.

Stochastic Games

  • Include chance nodes in game tree (e.g., dice rolls in Backgammon).
  • EXPECTIMINIMAX: At chance nodes, take weighted average over outcomes.
  • EXPECTIMINIMAX(C) = sum_r P(r) * EXPECTIMINIMAX(RESULT(C,r))
  • Eval function must be linear in outcome probabilities (not just monotone).
  • Complexity: O(b^m * n^m) where n = number of distinct dice outcomes.

Partially Observable Games

  • Kriegspiel (blind chess): Cannot see opponent pieces. Maintain a belief state over possible board positions.
  • Card games (Bridge, Poker): Cards dealt randomly — model as stochastic games OR use Monte Carlo averaging over opponent hand deals.
  • Proper solution: Solve with belief-state search; averaging over clairvoyance is a common shortcut but sub-optimal.

Limitations

  • Alpha-beta still requires exponential time in worst case.
  • Eval functions can be exploited if opponent knows them.
  • MCTS can be weak in tactical/sharp positions without good rollout policy.
Ch.6 Constraint Satisfaction Problems (CSPs)

CSP Definition

  • A CSP is defined by: Variables X, Domains D, Constraints C.
  • Goal: find assignment satisfying ALL constraints (consistent & complete).
  • Constraint types: unary, binary, global (e.g., AllDiff), soft.
  • Example: Map coloring — variables = regions, domain = {R,G,B}, constraint = adjacent regions differ.

Constraint Graph

  • Nodes = variables, edges = binary constraints.
  • Graph structure reveals tractable sub-problems (trees are solvable in linear time).

Arc Consistency (AC-3)

  • Arc (X,Y) is consistent if for every value in D(X) there is some value in D(Y) satisfying the constraint.
  • AC-3 repeatedly removes values that violate arc consistency; adds arcs back to queue when domain reduced.
  • Complexity: O(cd^3) where c = number of arcs, d = domain size.
  • AC-3 does NOT guarantee a solution — just reduces domains.
AC-3: queue = all arcs while queue not empty: (X,Y) = dequeue if REVISE(X,Y): if D(X) is empty: return FAILURE add all (Z,X) to queue for Z in neighbors(X) - {Y}

Backtracking Search

  • Assign one variable at a time; backtrack when no legal value remains.
  • Basic algorithm: DFS with constraint checking at each step.
  • Forward checking: After assigning X, remove values from neighbors' domains that conflict. Detects failure earlier.
  • MAC (Maintaining Arc Consistency): Run AC-3 after each assignment — stronger than forward checking.

Variable/Value Ordering Heuristics

HeuristicDescriptionType
MRV (Minimum Remaining Values)Choose variable with fewest legal valuesVariable
Degree HeuristicChoose variable with most constraints on remaining unassigned variablesVariable (tie-break)
LCV (Least Constraining Value)Choose value that rules out fewest values in neighboring domainsValue

Conflict-Directed Backjumping (CBJ)

  • Instead of backtracking one level, jump back to the variable that caused the conflict.
  • Each variable maintains a conflict set — set of past variables responsible for domain wipeout.
  • More efficient than chronological backtracking on large problems.
  • Constraint learning / nogood recording: Record the minimal inconsistent set as a new constraint to avoid re-exploring.

Local Search for CSPs

  • Start with complete (but inconsistent) assignment; iteratively repair conflicts.
  • Min-conflicts heuristic: For the variable to change, pick value that minimizes number of violated constraints.
  • Very effective on large problems (e.g., solves n-Queens for n=10^6 in seconds).
  • Plateau problem: use sideways moves and random restarts.

Problem Structure & Efficiency

StructureAlgorithmComplexity
Tree (no cycles)Topological sort + directional arc consistencyO(n * d^2)

  • Value symmetry: If permuting values gives equivalent solutions, add symmetry-breaking constraints (e.g., fix one color = Red).
  • Cycle cutset: Smallest set of variables whose removal leaves a tree. Finding minimum cutset is NP-hard, but good heuristics exist.
Ch.7 Logical Agents & Propositional Logic

Knowledge-Based Agents

  • Maintain a Knowledge Base (KB) — set of sentences in formal language.
  • TELL: Add new sentence to KB. ASK: Query KB (inference).
  • Agent operates at two levels: knowledge level (what it knows) and implementation level (how it's stored).
  • Entailment: KB |= a if every model of KB is also a model of a.

Wumpus World

  • 4x4 grid; agent has: Breeze/Stench/Glitter percepts; can Grab/Shoot/Climb.
  • Stench adj. to Wumpus; Breeze adj. to Pit; Glitter means Gold in cell.
  • Used to illustrate knowledge-based reasoning under uncertainty.
  • Agent deduces safe cells by combining percepts with logical rules.

Propositional Logic — Syntax & Semantics

  • Atomic sentences: P, Q, R (True/False).
  • Connectives: NOT, AND, OR, -> (implies), <-> (biconditional).
  • Model: Assignment of True/False to all variables.
  • A sentence is valid (tautology) if true in ALL models; satisfiable if true in some model.

Logical Equivalences

NameEquivalence
De Morgan'sNOT(A AND B) = NOT A OR NOT B
De Morgan'sNOT(A OR B) = NOT A AND NOT B

Biconditional Elim.A <-> B = (A->B) AND (B->A)
ContrapositiveA -> B = NOT B -> NOT A
Double NegationNOT NOT A = A

Conjunctive Normal Form (CNF)

  • CNF = conjunction of clauses; each clause = disjunction of literals.
  • Conversion steps: (1) Eliminate <->, (2) Eliminate ->, (3) Push NOT inward (De Morgan), (4) Distribute OR over AND.
  • Every propositional sentence can be converted to CNF.

Resolution Inference (PL-RESOLUTION)

  • Resolution rule: From (A OR B) and (NOT B OR C), derive (A OR C).
  • To prove KB |= a: add NOT a to KB, convert to CNF, apply resolution until empty clause (contradiction) or no new clauses.
  • Empty clause = FALSE = contradiction = proof found.
  • PL-RESOLUTION is complete for propositional logic.
PL-RESOLUTION(KB, alpha): clauses = CNF(KB AND NOT alpha) new = {} loop: for each pair of clauses (Ci, Cj): resolvents = RESOLVE(Ci, Cj) if {} in resolvents: return TRUE (entailed) new = new UNION resolvents if new is subset of clauses: return FALSE clauses = clauses UNION new

Horn Clauses & Definite Clauses

  • Horn clause: At most one positive literal. (NOT A OR NOT B OR C) = A AND B -> C.
  • Definite clause: Exactly one positive literal.
  • Fact: Single positive literal (no body).
  • Horn clauses allow efficient inference via forward/backward chaining in linear time.

Forward Chaining (PL-FC-ENTAILS?)

  • Data-driven: start from known facts, fire rules whose premises are satisfied.
  • Add conclusions to KB; repeat until query is in KB or no new facts.
  • Sound & Complete for Horn KBs.
  • Complexity: O(n) — each symbol added at most once.
PL-FC-ENTAILS?(KB, q): count[c] = number of symbols in premise of each clause c inferred = {} (all False) agenda = [all known facts in KB] while agenda not empty: p = agenda.pop() if p == q: return TRUE if not inferred[p]: inferred[p] = True for each clause c with p in premise: decrement count[c] if count[c] == 0: agenda.push(HEAD(c)) return FALSE

Backward Chaining

  • Goal-driven: start from query, find rules that conclude it, recurse on premises.
  • Uses depth-first search; avoids recomputing already-proved goals.
  • Sound & Complete for Horn KBs.
  • More efficient than forward chaining when only a few goals are needed.
  • Basis of logic programming (Prolog).
Ch.8 First-Order Logic (FOL)

Why FOL?

  • Propositional logic lacks expressiveness — can't talk about objects or relationships.
  • FOL adds: objects (constants), relations (predicates), functions, and quantifiers.
  • Commitment: ontological (objects, relations, facts exist) + epistemological (agent's beliefs).

Syntax

ElementDescriptionExample
ConstantRefers to a specific objectJohn, 2, KingJohn
VariableRanges over objectsx, y, z
PredicateRelation between objectsLoves(John,Mary), On(x,y)
FunctionMaps objects to an objectLeftLeg(John)
TermConstant, variable, or functionLeftLeg(x)
Atomic sentencePredicate applied to termsBrother(John,Richard)

Quantifiers

  • Universal (FOR ALL x): True if predicate holds for all objects. Typically used with implication: ForAll x: King(x) -> Mortal(x).
  • Existential (EXISTS x): True if predicate holds for some object. Typically used with AND: Exists x: Crown(x) AND OnHead(x,John).
  • Duality: ForAll x P(x) = NOT EXISTS x NOT P(x)
  • Scope & Nesting: Order matters — ForAll x Exists y Loves(x,y) differs from Exists y ForAll x Loves(x,y).
  • Equality: x = y means x and y refer to the same object.

Semantics & Models

  • A model specifies: a domain of objects, and interpretations of constants, functions, and predicates.
  • Unique-names assumption: Different constants = different objects (used in databases).
  • Closed-world assumption (CWA): Any fact not stated is assumed false.
  • Domain closure: No objects other than those named.
  • These three = database semantics. FOL by default uses none of them.

Kinship Domain

  • Mother(x) = y <-> Female(y) AND Parent(y,x)
  • Sibling(x,y) <-> x != y AND Exists p: Parent(p,x) AND Parent(p,y)
  • Grandparent(x,y) <-> Exists z: Parent(x,z) AND Parent(z,y)
  • Ancestor(x,y) <-> Parent(x,y) OR Exists z: Parent(x,z) AND Ancestor(z,y)
  • Shows recursion (Ancestor) and negation in FOL.

Numbers & Sets in FOL

  • Peano axioms: 0 is a number, Succ(x) gives the next number; addition and multiplication defined recursively.
  • Sets: Defined via Member predicate, intersection, union, subset axioms.
  • Lists: Nil (empty), Cons(h,t), Head, Tail analogous to LISP.

Wumpus World in FOL

  • Percepts: Percept([Stench,Breeze,None],1)
  • ForAll t: Stench(t) -> Exists s: Wumpus(s) AND Adjacent(s, Location(t))
  • FOL can express time-indexed facts and rules about the world more cleanly than propositional logic.

Electronic Circuits Domain

  • Types: Circuit, Gate, Terminal, Signal (0 or 1), Type (AND/OR/XOR/NOT).
  • Key predicates: Connected(t1,t2), Signal(t,s), GateType(g,t), Input(g,n,t), Output(g,n,t).
  • Axioms: connectivity symmetry, signal propagation through each gate type, uniqueness of output signal.
  • Example query: "Is output terminal 1 of circuit C1 connected to input terminal 1 of gate G2?"
  • Illustrates how FOL handles structured domains with objects and relations.
